Test Automation

რეგისტრაცია დასრულებულია.

კურსის აღწერა

რეგისტრაციის დაწყება: 10/09/24
რეგისტრაციის დასრულება: 22/09/24
ლექციების დაწყება: ოქტომბრის თვეში

ტესტირების ავტომატიზაცია გულისხმობს სპეციალიზებული პროგრამული ინსტრუმენტებისა და წინასწარ გაწერილი ნაბიჯების გამოყენებით პროგრამული უზრუნველყოფის ფუნქციონალის შემოწმებას. ამასთანავე, ტესტირების ავტომატიზაცია წარმოადგენს პროგრამული უზრუნველყოფის სასიცოცხლო ციკლის განუყოფელ ნაწილს.


კურსის ფარგლებში სტუდენტი შეისწავლის პროგრამულ ენებს და ინსტრუმენტს, როგორიცაა Java, Selenium, Rest Assured და ა.შ. მონაწილეები შეისწავლიან ასევე ტესტ დიზაინის საუკეთესო პრაქტიკებს, ტესტირების ჩარჩოს იმპლემენტაციას და ტესტების უწყვეტ ინტეგრაციას pipeline -ებში.


ტესტირების ავტომატიზაციის კურსის მიზანია ინდივიდების აღჭურვა იმ ცოდნითა და უნარებით, რომლებიც საჭიროა ტესტირების პროცესების გასაუმჯობესებლად, ადამიანური შეცდომების შესამცირებლად და პროგრამული უზრუნველყოფის ხარისხის გასაუმჯობესებლად ტესტირების ავტომატიზაციის გზით.

კურსის დასრულების შემდგომ სტუდენტებს შეეძლება ეჯაილ გუნდში Test Automation Engineer-ად მუშაობა.


კურსის გავლის შემდეგ შეგეძლება:

ავტომატური ტესტების შექმნა როგორც ვებ, მობაილ და დესკტოპ აპლიკაციებისთვის,ასევე Rest და Soap API-სთვის

დიზაინ პატერნებისა და საუკეთესო პრაქტიკების დაცვით პროექტის მართვა
პერფორმანს ტესტირების პროექტის აწყობა და მართვა
ავტომატური ტესტების ინტეგრაცია pipeline-ში

ვისთვისაა კურსი?

კურსის სამიზნე აუდიტორია მოიცავს 18 წლიდან ინდივიდების ფართო სპექტრს, რომლებიც ისწრაფვიან ისწავლონ ტესტირების ავტომატიზაცია.

მათ შორისაა: კომპიუტერული მეცნიერების სტუდენტები, თვითნასწავლი დეველოპერები,QA Manual ტესტერები (პროფესიონალები, რომლებიც ამჟამად მუშაობენ პროგრამული აპლიკაციების ტესტირების მიმართულებით, ამასთანავე აქვთ პროგრამირების საბაზისო ცოდნა და სურთ განვითარდნენ ტესტირების ავტომატიზაციის მიმართულებით) Career Switchers (პროფესიონალები, რომლებიც ამჟამად არიან სხვა სფეროებში, მაგრამ აქვთ დიდი ინტერესი გადართონ თავიანთი კარიერა ტესტირების ავტომატიზაციის მიმართულებით), არატექნიკური პირები (ადამიანები, რომლებსაც არ აქვთ ფორმალური ტექნიკური განათლება, მაგრამ აქვთ დიდი სურვილი და მოტივაცია ისწავლონ ტესტირების ავტომატიზაცია).

კურსის წინაპირობა: Java პროგრამული ენის საფუძვლები, OOP-ის პრინციპების თეორიული ცოდნა.

კურსის შინაარსი

შესავალი ტესტირების ავტომატიზაციაში

ტესტირების ავტომატიზაციის როლი პროგრამული უზრუნველყოფის განვითარების სასიცოცხლო ციკლში.

Java პროგრამირება ტესტირების ავტომატიზაციისთვის

Java პროგრამირების ენის ძირითადი კონცეფციები, OOP პრინციპები ეფექტური ავტომატური ტესტების შესაქმნელად.

ვებ აპლიკაციების ტესტირება Selenium/Selenide-ით

ვებ აპლიკაციებისთვის ავტომატური ტესტირების ფუნდამენტები Selenium/Selenide-ის გამოყენებით.

API ტესტირება Rest Assured - ით

API ტესტირების მნიშვნელობა, Rest და SOAP API - ების გარჩევა. Rest Assured-ის პრაქტიკული იმპლემენტაცია API ტესტირების ავტომატიზაციისთვის.

დიზაინის პატერნები ტესტირების ავტომატიზაციისთვის

დიზაინ პატერნების ფუნდამენტები და მათი შესაბამისობა ტესტირების ავტომატიზაციაში, მოქნილი და შენარჩუნებადი პროექტის ჩარჩოს მომზადება. Performance ტესტირება JMeter-ით

Performance ტესტირების მიმოხილვა და მისი მნიშვნელობა. პრაქტიკული მაგალითები JMeter-ის გამოყენებით.

ტესტების უწყვეტი ინტეგრაცია

CI/CD პრინციპების გააზრება პროგრამული უზრუნველყოფის შემუშავებაში. ავტომატური ტესტების ინტეგრაცია pipeline - ში Jenkins-ის გამოყენებით.

მობილური აპლიკაციის ტესტირება Espresso-თი

მობილური აპლიკაციების სპეციფიკის განხილვა, პრაქტიკული მაგალითები Espresso-ს გამოყენებით Android აპლიკაციებისთვის.

დესკტოპ აპლიკაციის ტესტირება TestComplete-ით

დესკტოპ აპლიკაციების სპეციფიკის განხილვა, პრაქტიკული მაგალითები TestComplete-ს გამოყენებით.

ვინ მასწავლის?

კურსის ტრენერი - ალინა ტყაბლაძე

ალინა ინფორმაციული ტექნოლოგიების სამაგისტრო პროგრამის კურსდამთავრებულია. იგი ცხრა წელზე მეტია, რაც თიბისი ბანკის ჯგუფის წევრია. 2019 წლიდან ალინა ტესტირების ავტომატიზაციის სფეროშია დასაქმებული, ამჟამად იგი ტესტირების ავტომატიზაციის ინჟინერი/ასოცირებული პარტნიორია. ალინა ასევე ჩართულია აშშ-ს Emory უნივერსიტეტის ციფრული პროდუქტების ტესტირებაში. 2020 წლიდან აქტიურად მოღვაწეობს განათლების სფეროში ისეთ უნივერსიტეტებში, როგორიცაა: კავკასია, BTU. თუმცა, პირველი ნაბიჯები განათლების სფეროში მან სწორედ თიბისი IT აკადემიაში გადადგა, სადაც 2020 წელს საქართველოში პირველი სრულყოფილი ტესტირების ავტომატიზაციის კურსის დაარსებაში მიიღო მონაწილეობა.

კურსის ტრენერი - ლაშა მოსახლიშვილი

ლაშა ინფორმაციული ტექნოლოგიების მაგისტრია. იგი ერთ-ერთი პირველი იყო, ვინც თიბისი ბანკის გუნდს შემოუერთდა ტესტირების ავტომატიზაციის მიმართულებით. ამჟამად იგი ტესტირების ავტომატიზაციის ინჟინერი/ასოცირებული პარტნიორის პოზიციაზეა დასაქმებული. ლაშა უკვე ორი წელია, რაც თავის ძირითად პროფესიას ტრენერის საქმიანობას უთავსებს თიბისი IT აკადემიაში, სადაც სტუდენტებს მობაილ აპლიკაციების ტესტირებას ასწავლის.

კურსის ტრენერი - ცოტნე აბურჯანია

ცოტნე ბიზნესისა და ტექნოლოგიების უნივერსიტეტის (BTU) ინფორმაციული ტექნოლოგიების კურსდამთავრებულია. იგი მესამე წელია, რაც თიბისი ბანკის ჯგუფს შემოუერთდა და ამჟამად ტესტირების ავტომატიზაციის ინჟინერია. ცოტნემ თავისი კარიერა სწორედ თიბისი IT აკადემიის Test Automation კურსით დაიწყო. მან 2021 წელს წარმატებით დაამთავრა კურსი და დასაქმდა ბანკში. ცოტნე უკვე 1 წელია, რაც Java პროგრამული ენის განხრით თავის ცოდნას და სამუშაო გამოცდილებას უზიარებს სტუდენტებს.

კურსის ტრენერი - თამარ მუმლაძე

თამარი თბილისის სახელმწიფო უნივერსიტეტის, კომპიუტერული მეცნიერებების მიმართულების კურსდამთავრებულია. აწმყოში იგი სწავლას ინფორმაციული სისტემების მართვის (DevOps) სამაგისტრო პროგრამაზე განაგრძობს. თამარი 3 წელია, რაც თიბისი ბანკის ჯგუფის ნაწილია, ამჟამად კი ტესტირების ავტომატიზაციის ინჟინერი/ასოცირებული პარტნიორის პოზიციაზეა დასაქმებული. თავის სწავლას/სამსახურებრივ საქმიანობას თამარი უკვე 1 წელზე მეტია, რაც ტრენერის მოვალეობებს უთავსებს თიბისი IT აკადემიაში.

კურსის ტრენერი - ირინა ინაშვილი

აირინა ინფორმაციული ტექნოლოგიების საბაკალავრო, ხოლო შემდგომ კი ინფორმაციული სისტემების მართვის (DevOps) სამაგისტრო პროგრამის კურსდამთავრებულია. იგი უკვე 5 წელია, რაც თიბისი ბანკის ჯგუფს შემოუერთდა. ამჟამად ირინა ტესტირების ავტომატიზაციის ინჟინერი/ასოცირებული პარტნიორია. იგი უკვე მეშვიდე მიღებაა, რაც თავის ცოდნას, სამუშაო გამოცდილებას და საუკეთესო პრაქტიკებს თიბისი IT აკადემიაში უზიარებს სტუდენტებს.

კურსის ტრენერი - თინათინ მჟავანაძე

თინათინი თბილისის სახელმწიფო უნივერსიტეტის ბიზნეს ადმინისტრირების სამაგისტრო პროგრამის კურსდამთავრებულია. იგი 5 წელია, რაც თიბისი ბანკის ჯგუფის წევრია, ხოლო 3 წელზე მეტია, რაც ტესტირების ავტომატიზაციის სფეროშია დასაქმებული. ამჟამად თინათინი ტესტირების ავტომატიზაციის ინჟინერი/ასოცირებული პარტნიორის პოზიციაზეა დასაქმებული. თინათინი მესამე წელია, რაც თავის პოზიციას აკადემიურ საქმიანობას უთავსებს თიბისი IT აკადემიაში. იგი თავის ცოდნას, სამუშაო გამოცდილებას და საუკეთესო პრაქტიკებს აკადემიის გარეთ, ასევე კავკასიის უნივერსიტეტში ლექტორის პოზიციაზეც აზიარებს.

ხშირად დასმული კითხვები

რა არის თიბისი IT აკადემიაში სწავლის ღირებულება?

თიბისი IT აკადემიის აბსოლუტურად ყველა კურსი 100%-ით დაფინანსებულია თიბისი ბანკის მიერ. შესაბამისად, კურსზე წარმატებით ჩარიცხვის შემთხვევაში სტუდენტს არ უწევს არანაირი საფასურის გადახდა.

რამდენ ხანს გრძელდება სწავლა თითოეულ კურსზე?

თიბისი IT აკადემიის კურსების ხანგრძლივობა მერყეობს სამი თვიდან ექვს თვემდე. დეტალური ინფორმაცია კურსის ვადების შესახებ შეგიძლიათ იხილოთ უშუალოდ კურსების აღწერის გვერდზე.

რამდენი სტუდენტის მიღება ხდება კურსზე?

კურსზე ადგილების რაოდენობა განსხვავებულია და მერყეობს 20-დან 30-მდე.

რა ეტაპებია გასავლელი კურსზე მოსახვედრად?

იმისათვის, რომ სტუდენტი ჩაირიცხოს თიბისი IT აკადემიის რომელიმე კურსზე საჭიროა შემდეგი ეტაპების გავლა:
1. რეგისტრაცია ვებ გვერდზე მითითებულ ვადებში
2. უნარების ონლაინ ტესტირება
3. ტექნიკური ტესტირება
4. HR სამოტივაციო ვიდეოს ატვირთვა
5. ტექნიკური გასაუბრება
ეტაპები შეიძლება განსხვავდებოდეს კურსის მიხედვით, მაგალითად, ზოგ შემთხვევაში, ამოღებულია უნარების ტესტირება. შესაძლოა ტექნიკური ტესტირება ჩანაცვლებული იყოს ტექნიკური დავალებით და ა.შ.

შემიძლია თუ არა რამოდენიმე კურსზე რეგისტრაცია?

დიახ, სტუდენტს შეუძლია დარეგისტრირდეს რამოდენიმე კურსზე და მიიღოს მონაწილეობა ერთზე მეტი კურსის შერჩევაში. თუმცა, აკადემიის შიდა პოლიტიკიდან გამომდინარე, აპლიკანტს შეუძლია მხოლოდ და მხოლოდ ერთ კურსზე სწავლა. შესაბამისად, იმ შემთხვევაში თუ აპლიკანტი ორი ან მეტი კურსის შერჩევის ეტაპს წარმატებით გაივლის მას მოუწევს აირჩიოს კონკრეტულად რომელი მიმართულებით უნდა გააგრძელოს სწავლა.

აკადემიის კურსებზე სწავლა ონლაინ რეჟიმში ტარდება თუ ფიზიკურად?

თიბისი IT აკადემიაში სწავლა ტარდება სრულიად ონლაინ.
